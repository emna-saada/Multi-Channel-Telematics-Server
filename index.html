Parameter effect

Inter-arrival time T. A smaller value of T means calls arrive more frequently, so the server fills up faster.
Service duration D. A larger value of D keeps channels occupied longer, which raises the risk of blocking.
Capacity N. Increasing the number of channels allows the server to absorb more traffic before saturation appears.

Interpretation panel

Problem. The server can process up to N simultaneous calls. There is no waiting queue, so excess demand is lost immediately.
Model. Inter-arrival times and service durations are exponential, leading to an M/M/N/N loss system.
Implementation. The simulation engine tracks arrival events, the first available channel, release times, state occupancy, and rejections.
Dashboard role. The room schema explains the phenomenon visually, while the charts verify it through utilization, blocking, convergence, and state probabilities.
